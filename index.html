    이전와 이후 벨류를 let preValue와 let nextValue로 정의합니다.

    셀렉터가 클릭되면 해당 셀렉터에 들어있는 옵션들을 querySelectorAll을 사용해 노드리스트로 가져옵니다.

    이렇게 가져온 옵션 노드리스트를 forEach로 돌립니다.

    각각의 옵션에서 구조분해를 사용하여 정의합니다.

    const { selected } = option;

    selected를 if문의 조건으로 사용하여 참일경우 옵션의 벨류를 뽑아서 preValue에 집어넣습니다.

    새로운 옵션을 클릭하여 옵션의 selected가 바뀔때 선택한 대상을 가져옵니다.

    e.target

    가져온 대상 내의 옵션들을 querySelectorAll로 가져옵니다.

    preValue값을 가져왔을때와 마찬가지의 방법으로 value값을 가져와 nextValue 에 집어 넣습니다.

    모든 셀렉터중에서 preValue와 같은 옵션 값이 있는것을 찾기위해 forEach로 각각의 셀렉터들을 돌리고 selected=ture 인 옵션들의 밸류를 가져와 nextValue 와 값이 같아 선택옵션이 중복상태에 놓일경우 selected = false로 바꿔주고 preValue와 같은 벨류를 가진 옵션을 찾아 selected =true를 넣어줍니다.

    이로서 다른 셀렉터에서 중복된 옵션을 선택하면 기존의 옵션을 변경되는 셀렉터의 기존옵션으로 바꾸어주는 로직이 완성됩니다.

    로컬을 활용한 최근접속기록 적용 사이트



    메인페이지 : TeProGames

    서브페이지 : 게시판 페이지



    TeProGames : 각 게임을 클릭하면 로컬에 저장 되고 최근 방문한 게임에 보이게 된다.

    게시판 : 글을 작성하면 메인페이지에서 제목을 볼 수 있다.

    TeProgames 최근접속게임 로직

    제이슨 파일에서 비동기로 데이터를 받아와 해당 데이터를 대괄호표기법을 사용해 제이슨 객체속 데이터들을 createEliment로 생성한 li에 innderHtml를 활용하여 집어넣어 게임아이템Li들을 생성합니다.

    이 생성된 li에 addEventlistener로 클릭이벤트를 생성하여 클릭시 최근방문한 게임 리스트에 append 매소드를 사용하여 추가되도록 만듭니다.

    이때 최근방문기록칸에 아이템이 기존의 5개에서 6개로 늘어나게 되므로 removeChild를 사용하여 제일 마지막 아이템을 game_items[game_items.length - 1] 형태로 length를 이용해 마지막 인덱스를 잡아 잡아 제거해줍니다.

    위와같이 아이템이 추가될때 추가되는 아이템내의 벨류들을 제이슨에서 가져올때의 역순으로 변수에 넣고 객체에 집어 넣습니다.

    이 생성된 객체를 로컬에 올려두고 페이지가 새로고침 될때 로컬에서 불러와 최근방문기록에 li아이템으로 만들어 생성하여 집어넣습니다.

    D-day를 정하고 매일 할일을 작성하는 웹앱





    캘린더 로직

    Date.now를 활용하여 받아온 값으로 let에 year month day를 객체로 받아옵니다.

    받아온 값에서 버튼을클릭하면 day가 1씩 오르도록 합니다. 달별 최대일수를 계산하는 함수를 만들어 버튼 클릭시에 달별 최대 일수가 자동으로 계산되어 변수에 들어가도록 만듭니다.

    day가 최대일수를 넘기면 day를 1로 바뀌게 만듭니다.
    day가 1보다 낮아지면 day를 최대일수로 바뀌게 만듭니다.

    이렇게 바뀐 day를 month year와 함께 화면에 뿌려주고 로컬에 업로드 합니다.

    화면을 새로 시작할때 로컬에서 받아온값을 변수에 넣어주고 화면에 뿌려줍니다.

    sass를 사용하여 제작한 PWA 웹앱 사이트 입니다.

    createElement를 사용하여 div 엘리먼트를 만들고 해당 div엘리먼트에 classList.add를 사용해 클래스를 넣어주고 innerHTML매소드를 사용해 내부에 코드를 넣어주되 채팅벨류가 들어가야할 부분에 템플릿 리터럴을 사용하여 벨류값을 변수로 받아와 들어오도록 만듭니다.
